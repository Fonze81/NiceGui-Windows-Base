# ▶️ Execution Modes

This document explains the current ways to start the **NiceGUI Hello World** application.

---

## 🧭 Available modes

| Mode                | Command                                 | Main purpose                              |
| ------------------- | --------------------------------------- | ----------------------------------------- |
| Pyproject command   | `nicegui-hello-world`                   | Normal execution after editable install   |
| Module execution    | `python -m nicegui_hello_world`         | Validate package execution                |
| Direct script       | `python src\nicegui_hello_world\app.py` | Quick diagnostic execution                |
| Development runner  | `python dev_run.py`                     | Browser mode with reload                  |
| Packaged executable | `.\dist\nicegui-hello-world.exe`        | Windows executable generated by packaging |

---

## 🖥️ Normal native execution

```powershell
nicegui-hello-world
```

Expected behavior:

- native mode enabled;
- reload disabled;
- startup source reported as `pyproject command`.

Expected message:

```text
Initializing NiceGUI Hello World from pyproject command in native mode with reload inactive.
```

---

## 📦 Module execution

```powershell
python -m nicegui_hello_world
```

This uses:

```text
src\nicegui_hello_world\__main__.py
```

Expected message:

```text
Initializing NiceGUI Hello World from module in native mode with reload inactive.
```

---

## 📄 Direct script execution

```powershell
python src\nicegui_hello_world\app.py
```

Expected message:

```text
Initializing NiceGUI Hello World from script in native mode with reload inactive.
```

This is useful for diagnostics. For regular use, prefer `nicegui-hello-world`.

---

## 🌐 Browser development mode

```powershell
python dev_run.py
```

`dev_run.py` calls:

```python
main(development_mode=True)
```

Then `main(...)` selects:

```python
if development_mode:
    native_mode = False
    reload_enabled = True
else:
    native_mode = True
    reload_enabled = False
```

Expected message:

```text
Initializing NiceGUI Hello World from dev_run.py in web mode with reload active.
```

Use this mode while editing the interface because browser mode with reload gives faster visual feedback.

### Windows multiprocessing guard

`dev_run.py` must keep this guard:

```python
if __name__ in {"__main__", "__mp_main__"}:
    main(development_mode=True)
```

NiceGUI reload mode can use multiprocessing on Windows. The internal `__mp_main__` name is not shown in the startup message; the user-facing source remains `dev_run.py`.

## 📦 Packaged executable

After packaging:

```powershell
.\dist\nicegui-hello-world.exe
```

Expected message:

```text
Initializing NiceGUI Hello World from package in native mode with reload inactive.
```

---

## 🖨️ Startup source detection

Startup source detection is implemented in:

```text
src\nicegui_hello_world\app.py
```

| Source              | How it is identified                                  |
| ------------------- | ----------------------------------------------------- |
| `dev_run.py`        | `development_mode=True`                               |
| `package`           | `is_packaged()` checks PyInstaller runtime state      |
| `pyproject command` | command created by `[project.scripts]` starts the app |
| `module`            | `python -m nicegui_hello_world` starts the app        |
| `script`            | `src\nicegui_hello_world\app.py` starts directly      |

---

## 🧭 Which mode should I use?

| Scenario                   | Recommended command                     |
| -------------------------- | --------------------------------------- |
| Normal local usage         | `nicegui-hello-world`                   |
| Validate package execution | `python -m nicegui_hello_world`         |
| Quick diagnostic run       | `python src\nicegui_hello_world\app.py` |
| Develop UI with reload     | `python dev_run.py`                     |
| Test packaged output       | `.\dist\nicegui-hello-world.exe`        |

---

## 🧊 Packaged execution and multiprocessing

The packaged executable uses the normal application entry point in `app.py`.

For packaged execution, `app.py` should keep a standard `__main__` guard with `freeze_support()`:

```python
if __name__ == "__main__":
    freeze_support()
    main()
```

The `__mp_main__` guard should not be added to `app.py` because packaged execution should not behave like development reload mode.

The `__mp_main__` guard remains only in `dev_run.py`, where it supports NiceGUI reload mode on Windows.

---

## 🖨️ Startup message in the page

`app.py` builds the startup message once in `main(...)`.

The same message is used for:

- terminal output through `print(...)`;
- the NiceGUI page, displayed after `Hello, NiceGUI!`.

This avoids duplicated formatting logic and keeps the terminal diagnostics aligned with the visible UI.
